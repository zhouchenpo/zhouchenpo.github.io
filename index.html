装箱与开箱

从int到Integer可以手动装箱,也可以自动装箱。

事实上，在IDEA中，前者还会被警告Unnecessary boxing，并且会被评论为deprecated。手动装箱并没有必要。IDEA：你这种傻逼也配开发java？

在与int类型进行比较与运算时，Integer会被自动开箱。当然也可以用Integer.intValue()手动开箱。不过IDEA依然不喜欢这种做法，认为其时Unnecessary的。

在做基本操作时手动开箱都是没有比较的，那么再复杂一点的操作呢？

原本以为IDEA会很高兴地让我自己开箱，但是没有。减法好像也不是很复杂。

再看mapToInt()操作。

虽然i->i.intValue的映射非常好理解，但是依旧不是非必要开箱。因为依然会自动开箱。

但当使用诸如arrayList.remove()这类函数时情况就不同了。

new吗

如果new两个对象，那么他们一定是不同对象。

但是IDEA会警告让我们不要手动装箱。那么如果都不手动装箱呢？

网上大家都说在[-128,127]范围内的Integer会进行缓存，下次再声明会直接引用。

当对其中一个进行修改后，两者又为不同的引用了。

当数值不处于[-128,127]时，两者同样不会为相同引用。

Java中的位操作

Posted on 2020-04-28 | In algorithm

位操作无非四种，取反~，按位与&，按位或|，按位异或^。另外还可以加上>>与<<。

但是利用位操作可以简化很多操作。直接看题。

统计二进制中1的个数

假设一个二进制数n=……10……0，那么n-1=……01……1，则有n&(n-1)=……00……0即可以消去最低位的1。

判断是否为2的幂

当然最简单的就是利用上面的统计1方法。因为2的幂的二进制一定为10……0的形式。

数组中出现一次的两个数字

给定一个数组，数组中有两个数字只出现了一次，其他数字都出现了两次，找出出现一次的两个数字。

看到题目想都不用想HashMap了。但是提交一看太慢了。

可以使用^操作。

将所有数字全部^运算后的结果就是出现一次的两个数的^。为了分别得到两个数，就要将整个数组分成两组，出现一次的两个数组在不同组内，每组内除了出现一次的数字外其他数字都要成对。

那么如何分组呢？

假设^操作后的结果是001111000。1代表该位两数不同，0代表该位相同。不同位肯定是一个0一个1，那么与1进行&操作的结果就不一样了。在这个例子里，最简单的就是取最后一个1位。所以可以按照&1000==0进行分组。至于其他数字，由于肯定成对，相同数字肯定结果相同分在相同组内。

数组中出现一次的一个数字

给定一个数组，其中只有一个数字出现一次，别的数字都出现了三次。找出只出现一次的数字。

真的是百思不得其解。先看看答案的操作。

^操作可以去掉重复两次，出现一次与三次则需要维护另一个数据seenTwice进行区分。

位操作都是符合交换律的，所以不妨分析一个数连续出现三次的情况。

初始时，两个变量均为0。出现一次时，seeTwice显然还是0，因此出现一次的结果即为0^num，即为num本身。出现两次时，seeOnce后半部分seeOnce^num抵消为0，此时显然seeOnce也为0。但是seeTwice变为了num本身。出现三次时全部相消为0。那么显然再循环一次即出现4次，5次又可以通过该方法计数，因此题目需要规定出现三次与一次。

java中的关键字

Posted on 2020-04-21 | In Java Language

static

简单来说，static可以实现“无实例”访问Class里的字段与方法。例如Math.max()。

修饰字段与方法时，被修饰的字段与方法将被储存在Class内，所有实例的调用都指向同一字段或数据。因此如果对一静态字段进行修改，相同类下的不同实例的字段也发生变化。所以，更推荐用Class.method()或者Class.field直接访问静态字段与方法。

interface也可以定义静态字段，且只能定义静态字段。所有字段必须为static final。（Why?）如果没有定义为static final，编译器会自动加上。

注意到main函数也为static(Why?)。

final

修饰Class

被修饰的类将无法被继承。类中的字段是否为final均可，但是所有的方法都将被自动定义为final。

修饰方法

被修饰的方法不能够被重写。用final可以防止子类对方法进行修改。所有的private方法会被隐式指定为final。

修饰字段

对于基本数据类型，被修饰的字段将不能被重新赋值。而对于引用类型的变量，该变量不能再指向其他对象。

abstract

abstract类不能实例化，只能被继承。类中的abstract方法不实现，需要继承后在子类内重写。如果子类没有重写所有的abstract方法，那么该子类依旧为abstract类。

abstract不能修饰static final private方法，因为他们不能被继承。

abstract也不能修饰final类，因为final类无法被继承。

volatile

volatile修饰的变量是线程安全的。

在CPU处理指令的过程中，为了增加效率，会将内存中的数据读取到L1,L2缓存内再进行运算，但不知道什么时候会写回去。当一个线程对一个变量进行修改后，在其写回前，其他线程仍可以读取原先的变量值。

在volatile修饰后，JVM会向CPU发出lock指令，实现缓存一致性协议。一个线程将该变量修改数据会被回写到内存，CPU在发现自己的缓存中的数据被修改后，会将缓存中的数据失效，并从内存中重新读取该数据。
