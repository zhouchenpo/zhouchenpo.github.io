java中的关键字

Posted on 2020-04-21 | In Java Language

static

简单来说，static可以实现“无实例”访问Class里的字段与方法。例如Math.max()。

修饰字段与方法时，被修饰的字段与方法将被储存在Class内，所有实例的调用都指向同一字段或数据。因此如果对一静态字段进行修改，相同类下的不同实例的字段也发生变化。所以，更推荐用Class.method()或者Class.field直接访问静态字段与方法。

interface也可以定义静态字段，且只能定义静态字段。所有字段必须为static final。（Why?）如果没有定义为static final，编译器会自动加上。

注意到main函数也为static(Why?)。

final

修饰Class

被修饰的类将无法被继承。类中的字段是否为final均可，但是所有的方法都将被自动定义为final。

修饰方法

被修饰的方法不能够被重写。用final可以防止子类对方法进行修改。所有的private方法会被隐式指定为final。

修饰字段

对于基本数据类型，被修饰的字段将不能被重新赋值。而对于引用类型的变量，该变量不能再指向其他对象。

abstract

abstract类不能实例化，只能被继承。类中的abstract方法不实现，需要继承后在子类内重写。如果子类没有重写所有的abstract方法，那么该子类依旧为abstract类。

abstract不能修饰static final private方法，因为他们不能被继承。

abstract也不能修饰final类，因为final类无法被继承。

volatile

volatile修饰的变量是线程安全的。

在CPU处理指令的过程中，为了增加效率，会将内存中的数据读取到L1,L2缓存内再进行运算，但不知道什么时候会写回去。当一个线程对一个变量进行修改后，在其写回前，其他线程仍可以读取原先的变量值。

在volatile修饰后，JVM会向CPU发出lock指令，实现缓存一致性协议。一个线程将该变量修改数据会被回写到内存，CPU在发现自己的缓存中的数据被修改后，会将缓存中的数据失效，并从内存中重新读取该数据。
