int与Integer

Posted on 2020-05-09 | In Java Language

背景

在做leetcode时发现

并不能顺利进入if，然后仔细一想，Stack<Integer>里应该是Integer,换用equals()后结果正常。

由于在push()操作时都是直接push(int)，因此之前并没有对两者的差别有过多思考。事实上在之前没有出问题大多是JDK自动给我装箱开箱了。

装箱与开箱

从int到Integer可以手动装箱,也可以自动装箱。

事实上，在IDEA中，前者还会被警告Unnecessary boxing，并且会被评论为deprecated。手动装箱并没有必要。IDEA：你这种傻逼也配开发java？

在与int类型进行比较与运算时，Integer会被自动开箱。当然也可以用Integer.intValue()手动开箱。不过IDEA依然不喜欢这种做法，认为其时Unnecessary的。

在做基本操作时手动开箱都是没有比较的，那么再复杂一点的操作呢？

原本以为IDEA会很高兴地让我自己开箱，但是没有。减法好像也不是很复杂。

再看mapToInt()操作。

虽然i->i.intValue的映射非常好理解，但是依旧不是非必要开箱。因为依然会自动开箱。

但当使用诸如arrayList.remove()这类函数时情况就不同了。

new吗

如果new两个对象，那么他们一定是不同对象。

但是IDEA会警告让我们不要手动装箱。那么如果都不手动装箱呢？

网上大家都说在[-128,127]范围内的Integer会进行缓存，下次再声明会直接引用。

当对其中一个进行修改后，两者又为不同的引用了。

当数值不处于[-128,127]时，两者同样不会为相同引用。

还有其他的

虽然大家都说int的初始值是0,Integer的初始值是null。虽然拍拍脑袋确实是这样，但是不初始化变量并不能给我build。

Intger是int的包装类，别的基本数据类型同样有包装类。
